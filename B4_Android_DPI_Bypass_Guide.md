# B4 & ByeDPI — Полное руководство по обходу блокировок на Android (Версия 1.08)

> **Официальный гайд по тестированию, настройке и архитектуре** локального Android-приложения `myVPNproject` на базе концепций **B4 (Bye Bye Big Bro)** и **ByeDPI (ciadpi)**.

---

## 1. Как работает обход блокировок на устройстве (v1.08)

Это приложение **НЕ отправляет** ваш трафик на сторонние сервера и **НЕ является** классическим шифрующим VPN [1]. Обход DPI (Deep Packet Inspection) происходит **локально на процессоре вашего телефона** [1].

### Ключевой принцип обхода (TCP Segment Splitting & QUIC Drop):
1. **Блокировка QUIC (UDP 443 Drop)**: Популярные браузеры (Opera, Chrome) и приложения (YouTube) по умолчанию используют быстрый протокол QUIC поверх UDP [10]. В UDP нет механизма фрагментации, поэтому провайдерский DPI мгновенно видит заблокированный SNI и блокирует его. 
   **Решение**: Приложение блокирует QUIC, принудительно заставляя Opera откатиться на стандартный TCP/TLS [10].
2. **TCP-фрагментация (Active Split)**: Как только Opera отправляет TCP-пакет `TLS ClientHello` (запрос к сайту), наше приложение делит его на два сегмента (например, первый размером всего в 5 байт — до поля доменного имени) [3]. 
   DPI провайдера анализирует только первый пакет, не находит в нем заблокированного ключевого слова (SNI) и пропускает его [3]. Целевой веб-сервер склеивает фрагменты обратно и отдает страницу [3]!

## 2. Пошаговая инструкция по тестированию обхода блокировок

Если у вас ранее не открывался заблокированный сайт через браузер **Opera**:

### Шаг 1. Полное удаление конфликтующего пакета (Важно!)
Если при установке файла `v1.08` пишется *"Приложение не установлено, так как оно конфликтует с другим пакетом"*:
1. Вручную удалите предыдущую версию `myVPNproject` со своего телефона [2].
2. Установите новый APK-файл версии `1.08`.
3. *Все последующие обновления теперь будут устанавливаться поверх без удаления благодаря синхронизации ключей подписи на GitHub Actions [3]!*

### Шаг 2. Выбор приложений в интерфейсе
1. Запустите приложение **myVPNproject (v1.08)**.
2. Перейдите на вкладку **"Приложения"** (вверху экрана).
3. Теперь, благодаря интеграции системного разрешения `QUERY_ALL_PACKAGES`, вы увидите полный список всех установленных на вашем смартфоне приложений.
4. Найдите в списке и отметьте галочкой браузер **Opera** (а также любые другие браузеры/приложения, которые вы хотите протестировать).

### Шаг 3. Запуск обхода
1. Вернитесь на вкладку **"Инструкция"** в приложении.
2. Нажмите большую зеленую кнопку **"ЗАПУСТИТЬ VPN"** и подтвердите стандартный запрос Android на создание локального VPN-соединения.
3. Откройте выбранный браузер **Opera** и перейдите на заблокированный сайт. Сайт успешно откроется!
4. Все не отмеченные в списке приложения будут работать напрямую через обычное интернет-соединение на максимальной скорости вашего провайдера.

## 3. Архитектура и структура исходного кода (Справочник для ИИ)

Проект построен на базе современных стандартов Android-разработки [5]:
* **`BypassVpnService.kt`**: Нативная служба фонового VPN. Настраивает селективное туннелирование (Split Tunneling) только для выбранных пользователем пакетов приложений [1].
* **`PacketProcessor.kt`**: Выполняет разбор IP-пакетов в Userspace. Дропает QUIC/UDP и применяет активный TCP Split для TLS ClientHello [1].
* **`MainActivity.kt`**: Современный реактивный UI на **Jetpack Compose** с переключением вкладок (Инструкция / Список приложений с чекбоксами) и полной автодиагностикой.
* **`AndroidManifest.xml`**: Содержит строгие декларации фоновых служб для Android 14+ (`specialUse`) и право на сканирование установленных пакетов (`QUERY_ALL_PACKAGES`).
